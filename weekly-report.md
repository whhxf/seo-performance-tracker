# 独立站运营周报 — Agent 执行手册

> 本文档是 `seo-performance-tracker` Skill 的周报子模块。  
> 当用户说「生成周报」「帮我做独立站周报」「每周复盘」时，Agent 读取本文档按流程执行。

---

## 一、为什么需要固定周报？

独立站 SEO 是复利游戏——每周的微小变化累积成趋势。没有固定复盘，以下问题会被忽视：

- 排名骤降但 2 周后才发现（损失已经造成）
- 内容发布了但没被收录（白忙一场）
- 某篇文章突然爆量但没追加内链（机会浪费）
- CTR 持续偏低但从没优化过 Meta（显而易见的优化迟迟不做）

**周报的核心价值**：每周花 15 分钟，把「感觉」变成「数据」，把「随机发现」变成「系统追踪」。

---

## 二、7 大关键维度速查

每次周报必须覆盖以下 7 个维度，缺一不可：

```
维度 1：搜索可见性    → 我被看见了多少次？愿意点进来吗？
维度 2：自然流量质量  → 来的人是否真实有效？有没有留下？
维度 3：关键词排名动态 → 核心词在进步还是退步？
维度 4：页面内容表现  → 哪些内容在带流量？哪些在浪费曝光？
维度 5：转化漏斗      → 流量有没有变成商业价值？
维度 6：Bing 生态     → Bing 渠道是否健康（别只盯 Google）？
维度 7：内容发布节奏  → 本周计划执行了吗？下周优先做什么？
```

---

## 三、Agent 执行步骤（每次生成周报的固定流程）

### Step 0：确认参数

向用户确认（若对话中已明确则跳过）：
- 站点域名（如 `example.com`）
- 时间范围（默认：**本自然周 vs 上自然周**）

### Step 1：拉取数据（依次执行）

```bash
# 1. GSC 展示/点击/CTR/排名（本周7天）
node scripts/gsc-report.cjs <站点代号> 7

# 2. GSC 关键词维度（查询词列表）
node scripts/gsc-report.cjs <站点代号> 7 --queries

# 3. GA4 流量质量 + 页面表现
node scripts/ga4-report.cjs <站点代号> 7

# 4. GA4 转化数据（如已配置转化目标）
node scripts/ga4-report.cjs <站点代号> 7 --conversions

# 5. Bing Webmaster
node scripts/bing-report.cjs

# 6. 读取话题树发布计划（如存在）
# 读取 outputs/*/publish-schedule.md 获取发布进度
```

> 若某个数据源未配置（如未配置 GA4），跳过该步骤并在报告中注明"数据源未配置"。

### Step 2：分析数据，识别关键信号

在填写模板前，Agent 需先完成以下判断：

**搜索可见性预警检测：**
- [ ] 总展示环比下跌 > 15%？ → 标记 🔴 P0
- [ ] 总点击环比下跌 > 10%？ → 标记 🔴 P0
- [ ] 平均排名下跌 > 3 位？ → 标记 🟡 P1

**关键词异常检测：**
- [ ] 有排名骤降词（环比跌 > 5 位）？ → 标记为预警
- [ ] 有新触达词（上周无排名，本周进入 Top 50）？ → 标记为机会

**内容机会检测：**
- [ ] 有高展示低点击词（展示 > 500 且 CTR < 1%）？ → 加入 P1 建议
- [ ] 有高流量零转化页面？ → 加入 P1 建议

**内容进度检测：**
- [ ] 本周发布执行率是否 < 80%？ → 分析原因
- [ ] 有新发布页面超过 7 天未被收录？ → 建议手动提交

### Step 3：生成结构化周报

按以下模板填写，**所有 [占位符] 必须替换为真实数据**：

---

```markdown

> 生成时间：{timestamp} | 数据来源：GSC + GA4 + Bing Webmaster
```

## 四、周报健康度判断标准

| 信号 | 阈值 | Agent 响应 |
|------|------|------------|
| 展示量连续 2 周下跌 | > 15%/周 | 主动提示检查技术 SEO 或算法更新 |
| CTR 低于 1% | 全站平均 | 建议批量优化 Meta 标题/描述 |
| 核心词排名跌出 Top 20 | 任意核心词 | 立即分析竞品，建议内容更新 |
| 互动率低于 40% | GA4 Organic | 内容与搜索意图不匹配，需重写 |
| 新发布内容 14 天未收录 | 任意页面 | 手动提交 + 检查内链 |
| 转化率环比下跌 > 20% | — | 检查 CTA、落地页体验、表单 |
| Bing 爬取错误增加 | 任意新增 | 登录 Bing Webmaster 排查 |

## 六、常见问题

**Q：某个数据源没配置怎么办？**  
A：跳过该维度，在报告中注明「[数据源] 未配置，本节跳过」，其他维度正常生成。

**Q：首次运行没有上周数据怎么对比？**  
A：第一次无法环比，展示绝对值，注明「本周为基线数据」。

**Q：想只看某一个维度怎么办？**  
A：用户说「只看关键词排名」「帮我看看本周的转化情况」→ Agent 只执行对应维度，不触发完整周报流程。
